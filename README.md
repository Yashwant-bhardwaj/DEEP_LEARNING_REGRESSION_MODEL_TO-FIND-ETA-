## 📌 Abstract
This repository contains a complete, end-to-end Machine Learning research pipeline built with PyTorch to estimate the precise Time of Arrival (ETA) for buses. Predicting real-world transit times is a complex regression problem. This research simulates a smart bus tracking system's backend prediction logic, factoring in distance, traffic hours, weather conditions, and weekend variations to train a highly optimized deep neural network.

This module is designed to serve as the predictive intelligence layer for real-time GPS coordinate streaming applications.

---

## 🔬 Research Methodology & Pipeline

This project takes an empirical approach to model architecture, systematically testing various hyperparameters to find the absolute optimal configuration for regression. The pipeline automates the comparison of:
1. **Activation Functions** (ReLU, LeakyReLU, ELU)
2. **Learning Rates** (0.1 to 0.0001)
3. **Network Architectures** (Shallow, Medium, Deep)
4. **Hidden Layer Neurons** (16, 64, 128)
5. **Loss Functions** (MSE, MAE, Huber)

### 📊 Dataset Generation
Since real-world transit data is often proprietary, this pipeline includes a robust synthetic data generator that creates `5000` realistic transit records. The engine calculates base travel time (`Distance / Speed`) and applies multiple real-world friction layers:
* **Traffic Multipliers:** High congestion between `09:00-11:00` and `17:00-20:00`.
* **Weather Delays:** Normal, Light Rain (1.3x delay), Heavy Rain (1.6x delay).
* **Weekend Adjustments:** 30% probability of weekend traffic patterns.
* **Gaussian Noise:** Added to simulate random real-world anomalies.

## 📈 Experimental Results & Findings

After running the automated training pipeline across multiple epochs, the model evaluated the configurations based on Root Mean Squared Error (RMSE). Below are the final empirical results:

### 🏆 Optimal Configuration Discovered
| Parameter | Best Configuration | RMSE Score |
| :--- | :--- | :--- |
| **Activation Function** | `LeakyReLU` | 517.18 |
| **Learning Rate** | `0.01` | 621.44 |
| **Architecture Complexity**| `Deep` *(128 -> 64 -> 32 -> 16)*| 505.59 |
| **Layer Neurons** | `128` | 978.07 |
| **Loss Function** | `Huber Loss` *(SmoothL1Loss)* | **412.84** |

**Key Takeaways:**
1. **Huber Loss Outperformed MSE/MAE:** Because real-world traffic data contains sudden outliers (e.g., accidents), Huber Loss proved vastly superior to standard Mean Squared Error, achieving the lowest overall error (RMSE: 412.84).
2. **Network Depth Matters:** The `Deep` architecture learned the non-linear combinations of weather and rush-hour traffic much better than the `Shallow` network.
3. **LeakyReLU over ReLU:** Mitigated the "dying ReLU" problem during backpropagation, yielding better convergence.
